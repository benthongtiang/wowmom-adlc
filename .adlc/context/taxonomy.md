# Taxonomy — Retrieval Tag Vocabulary

This project's official vocabulary for retrieval tag dimensions utilized during specification-driven development sessions.

## component (narrow area)
Values identify specific architectural subsystems or views:
- `API/auth`
- `API/mothers`
- `API/groups`
- `API/applications`
- `API/admin`
- `UI/auth`
- `UI/mother`
- `UI/leader`
- `UI/admin`
- `services/notifications`

## domain (broad area)
High-level feature clusters:
- `auth`
- `discovery`
- `applications`
- `interviews`
- `roster`
- `analytics`

## stack (tech layers)
Technologies active across components:
- `react`
- `react-native`
- `nodejs`
- `express`
- `sql`
- `nosql`
- `markdown`

## concerns (cross-cutting dimensions)
Systemic quality parameters:
- `security`
- `capacity-enforcement`
- `state-consistency`
- `mobile-responsiveness`
- `notifications`

## tags (free-form)
Ad-hoc descriptors for precise context targeting. Examples include: `interview-scheduling`, `capacity-check`, `roster-masking`, `admin-approval`, `reapplication-cooldown`.
